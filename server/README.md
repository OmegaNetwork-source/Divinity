# Divinity Upgrade API

Backend that handles Divinity NFT upgrades. When a user pays USDC on the website, this server:
1. Verifies the USDC payment on-chain
2. Picks an Angel or Demon image from the uploaded manifest
3. Builds and uploads new metadata JSON to Arweave (Irys)
4. Updates the user's NFT on-chain via Metaplex `updateV1`

## Upgrade Tiers

| Tier | Price | Selection |
|------|-------|-----------|
| 1    | $5 USDC  | Random Angel or Demon |
| 2    | $10 USDC | User chooses Angel or Demon |
| 3    | $20 USDC | User chooses Angel or Demon |

## Prerequisites

- `burner.json` in the project root (update authority keypair)
- `upgrade_manifest.json` in the project root (generated by `build-upgrade-manifest.js`)
- Enough SOL in the burner wallet for Irys uploads (~0.001 SOL per upgrade metadata JSON)

## Setup

```bash
cd server
npm install
```

## Run

```bash
npm start
```

Runs on http://localhost:3001. Set `PORT` env var to change.

## Endpoints

### POST /api/upgrade
Submit an upgrade after USDC payment.

**Body:**
```json
{
  "tier": 1,
  "solanaWallet": "...",
  "nftMint": "...",
  "choice": "angel",
  "txSignature": "..."
}
```
- `tier`: 1, 2, or 3
- `choice`: Required for tiers 2 & 3 (`"angel"` or `"demon"`)
- `txSignature`: The USDC transfer transaction signature (verified on-chain)

**Response:**
```json
{
  "ok": true,
  "id": "...",
  "realm": "Angel",
  "imageUri": "https://gateway.irys.xyz/...",
  "metadataUri": "https://gateway.irys.xyz/...",
  "message": "Your NFT has been upgraded to Angel! Check your wallet."
}
```

### GET /api/health
Health check.

## Data

- `upgrades.json` — Records of all upgrades (auto-created)
- Images cycle through 2,500 Angels and 2,500 Demons via round-robin

## Production Deployment

Deploy to Railway, Render, or any Node.js host. Set these environment variables:
- `PORT` — Server port (default: 3001)
- `RPC_URL` — Solana RPC URL (defaults to Helius mainnet)

Make sure `burner.json` and `upgrade_manifest.json` are accessible to the server.
Then update `UPGRADE_API_URL` in `script.js` to point to your deployed server URL.
